Learn it in two-twos

The whole language on one page. Every script opens with wagwan and closes with say less — everything in between is up to you.

Open & close — wagwan / say less

Greet mans on the way in, dip politely on the way out.

wagwan

  big up "hello from the 6ix"

say less

Print — big up

Put anything on the console: text, numbers, lists, whatever.

big up "wagwan, world!"
big up 416
big up "2 + 2 is " + (2 + 2)

Variables — ting

Declare a ting with ting … is, update it with plain is.

ting city is "Toronto"
ting area is 416
area is area + 231
big up city + " runs on " + area

If / else — ahlie / nahfam

"Ahlie?" checks if it's true. "Nahfam" is what happens when it's not.

ahlie (temp < 0) {
  big up "bare cold out"
} nahfam ahlie (temp > 25) {
  big up "patio season"
} nahfam {
  big up "regular Toronto weather"
}

While — run it back

Keeps running it back while the condition stays true.

ting n is 3
run it back (n > 0) {
  big up n
  n is n - 1
}

For — two-twos

A counted loop, done in two-twos. Add by to set the step; count down by flipping the range.

two-twos i from 1 to 5 {
  big up i
}
two-twos j from 10 to 0 by 2 {
  big up j
}

Functions — big man ting

Define one with big man ting, hand the answer back with dash.

big man ting linkUp(name) {
  dash "wagwan " + name
}

big up linkUp("fam")

Loop control — dip / gwan

dip breaks out of the loop, gwan skips to the next round.

two-twos n from 1 to 10 {
  ahlie (n == 3) { gwan }
  ahlie (n == 6) { dip }
  big up n
}

Lists

Square brackets, indexed from 0. chuck adds to the end, bare counts.

ting squad is ["dot", "sauga", "vaughan"]
chuck(squad, "brampton")
big up squad[0]
big up "squad is " + bare(squad) + " deep"

Truth & nothing — styll / cap / ghost

True is styll, false is cap, and nothing at all is ghost.

ting realTalk is styll
ting rumour is cap
ting plans is ghost
ahlie (realTalk and not rumour) {
  big up "no cap detected"
}

Comments — chattin

Anything after chattin is just chatting — the interpreter ignores it.

chattin this line does nothing
big up "this one runs"  chattin this part doesn't

Input — pree

Ask the user for something. Comes back as text — use num() if you need a number.

ting name is pree("who dis?")
big up "wagwan " + name
ting age is num(pree("how old?"))
big up "next year you're " + (age + 1)

What's the deal?

6ix Lang is a real (toy) programming language: the code you type is run by a proper interpreter — a lexer, a recursive-descent parser, and a tree-walking evaluator — written in plain JavaScript, entirely in your browser. Nothing is sent to a server.

When you mess up, the error messages talk back in slang: forget the wagwan and it'll ask why you're just startin' to chat. Loops that run back forever get cut off before they freeze your tab, styll.

Made for jokes and for learning how interpreters work. Peep the source, fork it, and swap the keywords for your own city's slang if you want — the whole language lives in one readable file.
